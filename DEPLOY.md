# دليل النشر — منصّة الاختبارات الإلكترونية

نسخةٌ واحدة من الكود تعمل في الحالتين (محلّي على شبكة مؤسّسة، أو على الويب)؛
الفرق **إعدادات بيئة** فقط (ملفّ `.env`).

## متغيّرات البيئة
| المتغيّر | الوصف |
|---|---|
| `DATABASE_URL` | اتصال PostgreSQL (محلّي أو Neon). |
| `AUTH_SECRET` | نصّ طويل عشوائي (≥ 16 محرفاً) لتوقيع الجلسات. |
| `COOKIE_SECURE` | `true` على الويب (HTTPS)، `false` على الشبكة المحلّية (HTTP). غيابه = حسب البيئة. |

> **مهمّ:** على الشبكة المحلّية عبر HTTP يجب ضبط `COOKIE_SECURE=false` وإلا لن يُرسَل كوكي الجلسة فيفشل الدخول.

## أ) نشر محلّي على شبكة مدرسة/معهد (بلا إنترنت)
1. جهّز جهازاً خادماً دائم التشغيل واعرف عنوانه: `ipconfig` → مثلاً `192.168.1.50`.
2. ثبّت **Node.js 18+** و**PostgreSQL** عليه، وأنشئ قاعدة محلّية.
3. أنشئ `.env`:
   ```
   DATABASE_URL=postgresql://user:pass@localhost:5432/bacexam
   AUTH_SECRET=ضع_نصاً_طويلاً_عشوائياً
   COOKIE_SECURE=false
   ```
4. `npm install` ← `npm run db:push` ← `npm run db:seed` (أو `npx tsx prisma/demo.ts` لعالم تجريبي).
5. `npm run build` ثم `npm run start -- -H 0.0.0.0 -p 3000`.
6. افتح منفذ **3000** في جدار حماية Windows (وارد).
7. الطلاب على نفس الشبكة يفتحون `http://192.168.1.50:3000` ويدخلون بنافذة دورهم.
8. لإبقائه خدمةً تعيد التشغيل: NSSM / Task Scheduler / pm2.

## ب) نشر على الويب
1. قاعدة مُدارة مُحجَّمة (Neon مدفوع أو ما يكافئه) مع تجميع اتصالات.
2. `.env`:
   ```
   DATABASE_URL=...    # سلسلة Neon مع pooling
   AUTH_SECRET=نصّ_طويل_عشوائي
   COOKIE_SECURE=true
   ```
3. `npm install` ← `npm run db:push` ← `npm run db:seed`.
4. `npm run build` ثم `npm run start` خلف HTTPS (منصّة استضافة أو وكيل عكسي).
5. للتوسّع لاحقاً: نقل تخزين المرفقات لمزوّد كائني، إضافة كاش/CDN ومحدّد معدّل مشترك (Redis).

## بعد النشر
- فحص الصحّة: `GET /api/health` → `{ "ok": true, "db": "up" }`.
- الحساب الأول بعد البذرة: المدير العام `admin@example.com` / `Admin@123` (غيّر كلمته فوراً).
- المنصّة مُستثناة من فهرسة محرّكات البحث (`robots.txt`).
