# منصة الاختبارات الإلكترونية — تعليمات المشروع (CLAUDE.md)

> اقرأ هذا الملف قبل أي عمل. التزم به حرفياً. لا تعيد هندسة البنية ولا تغيّر المخطط دون إذن صريح.

## اللغة والواجهة
- ردّ دائماً بالعربية. كل الواجهات عربية، اتجاه RTL، متجاوبة للموبايل والحاسوب.
- مخاطبة حسب الجنس: مدير/مديرة، مدرّس/مدرّسة، طالب/طالبة، «مرحباً بك/بكِ».

## المعمارية (لا تُغيَّر دون إذن صريح)
- تطبيق Next.js 14 واحد (App Router) — واجهة + API. لا Docker، لا Redis، لا Nginx.
- TypeScript. Tailwind بدعم RTL. خطّا Tajawal وReem Kufi.
- PostgreSQL عبر Prisma (Neon مجاناً الآن، مدفوع عند الإطلاق).
- مصادقة JWT عبر `jose` في كوكي httpOnly، والجنس داخل الجلسة.
- تحقّق Zod في كل مسار API.
- تجاهل أي مقترح مناقض (NestJS/Django/Docker/Redis/S3/VPS/Kubernetes) إلا بطلب صريح.

## القرارات المثبّتة (احترمها)
- حقل `gender` على نموذج `User` لكل الأدوار، ومُضمَّن في الجلسة.
- علامة قابلة للتجاوز لكل اختبار عبر `pointsOverride` على `QuizNode`.
- المدرّس المُنشئ + المدير فقط يعدّلان بيانات الطالب وكلمة سرّه.
- نافذة توقيت واحدة لكل اختبار (availableFrom/Until + المحاولات + المهلة).
- تصحيح الإجابة القصيرة: تطبيع عربي (تشكيل/همزات/تاء مربوطة) + مطابقة دقيقة لا جزئية.
- جدول `AppSetting` لإعدادات المدير (الخط/الثيم).
- الدرجة تُخزَّن كنسبة من «الأسئلة الصالحة» لتُعاد الحسبة بأمان عند إلغاء سؤال.

## الأمان (إلزامي)
- لا تكشف أو تحتفظ ببيانات اعتماد قاعدة البيانات؛ إن ظهرت، نبّه فوراً واطلب إعادة تعيينها.
- تحقّق من الملكية والصلاحية في كل استعلام (المدرّس موارده، الطالب جلسته، ولي الأمر بيانات ابنه).
- لا تُرسل الإجابات الصحيحة للمتصفّح أثناء الأداء.
- فرض المهلة الزمنية على الخادم (وقت البدء من `ExamSession`)، لا على المتصفّح.
- خصوصية القُصّر: لا تُتح بيانات طالب لغير المخوّلين.

## أعراف الكود
- عميل Prisma واحد مشترك (singleton) في `src/lib/prisma.ts` — لا عميل لكل طلب.
- لفّ الكتابات المتعددة في معاملة واحدة (transaction).
- التصحيح يتم على الخادم دائماً.
- اختبارات آلية أولاً للأخطر: دالة التطبيع/التصحيح ودالة إعادة حساب الدرجات.

## طريقة العمل
1. اقترح النطاق بدقّة (ماذا سيُبنى وماذا لن يُبنى الآن) وانتظر الموافقة.
2. ابنِ خطوة واحدة قابلة للاختبار.
3. اسأل قبل أي تغيير في مخطط قاعدة البيانات.
4. بعد كل خطوة وضّح ما يلزم: npm install / db:push / db:seed / إعادة تشغيل.
5. قدّم تعليمات Windows/PowerShell. لا تكسر ما يعمل. أسئلة قليلة ومركّزة.

## بروتوكول الأفكار الطارئة
عند طرح فكرة جديدة أثناء خطوة جارية: لا توقف الخطوة. صنّفها:
(أ) تعديل بسيط آمن → نفّذه وأخبر. (ب) ميزة جديدة → أضِفها للـ Backlog ولا تبدأها قبل إذن.
(ج) تغيير معماري/مخطط → توقّف ونبّه وناقش. واعرض قائمة «أفكار مؤجّلة» في نهاية أي ردّ فيه فكرة مؤجّلة.

## الحالة الحالية (مكتمل ويعمل)
**الحلقة كاملة + إدارة + تعدّد مؤسّسات.** أهمّ ما بُني:

**الأساس:** Next.js 14 (App Router) + TS + Tailwind RTL + خط Cairo (متغيّر `--font-cairo`). مصادقة jose (كوكي httpOnly، الجلسة تحمل sub/role/gender). Prisma/Postgres (Neon). تحقّق Zod في كل مسار. عميل Prisma singleton.

**حلقة الطالب:** `grading.ts` (تطبيع عربي + مطابقة دقيقة + إعادة حساب) + 19 اختبار Vitest. الأداء (قائمة/بدء/إجابة/تخطّي/نتيجة) بتصحيح على الخادم، مهلة مفروضة من الخادم، **تخطّي مؤقّت** والعودة للمتخطّى آخر الاختبار، تصحيح فوري أو في النهاية حسب الإعداد. الدخول: صفحة بداية بـ**نوافذ حسب الدور** (طلاب/مدرّسون/مدراء/أولياء) — كل نافذة تُلمّح للمُعرّف المناسب، والخلفية موحّدة تحلّ بريد **أو** رمز الطالب **أو** رمز المدرّس **أو** الاسم، مع تحقّق النافذة بعد كلمة السر فقط (لا يكشف وجود الحساب).

**التصحيح اليدوي:** القصيرة تُصحَّح آلياً مبدئياً، والمقالية يدوية؛ كلاهما `needsReview` → الطالب يرى «بانتظار التصحيح» (تُخفى نتيجته النهائية)، والمدرّس يصحّح من صفحة الجلسة (قابل للتعديل دائماً) ويُعاد الحساب.

**بنك الأسئلة + المنهج:** أنواع: اختيار/صح-خطأ/قصيرة/**مقالي**. تشجير: المادة ← **وحدة (`Unit`)** ← فصل (`Chapter`) ← درس (`Concept`). المدرّس يبني المنهج (`/teacher/curriculum`) ويفلتر البنك بالتشجير + ترقيم صفحات. قفل بنية السؤال المُستخدَم.

**تكوين الاختبار والإسناد:** باني خطّي (اختيار/ترتيب/`pointsOverride`/إعدادات/نافذة توقيت/نشر) مع قفل المنشور/المُستخدَم. الإسناد: فردي/متعدّد بالاختيار/للكل/**بالرمز** (لصق رموز طلاب المؤسّسة)/**رمز الاختبار** (`accessCode` تسلسلي، انضمام الطالب به مع مفتاح تفعيل `allowCodeJoin`). منح محاولة إضافية/تصفير لكل طالب (`extraAttempts`).

**المتابعة:** نتائج المدرّس لكل اختبار + مراجعة جلسة كل طالب (`getSessionReview` مشترك).

**إدارة المدرّس لطلابه:** إنشاء/تعديل/كلمة سرّ، رمز طالب تسلسلي تلقائي، حقول (أب إجباري/أم/عنوان/هاتف الطالب)، تسجيل في المواد.

**تعدّد المؤسّسات:** `School` + `User.schoolId`. **مدير عام للمنصّة** (schoolId=null, isSuperAdmin) ينشئ المدارس/المعاهد ومديريها والمواد/الصفوف والحقول المخصّصة. **مدير مدرسة** (admin بـ schoolId) يدير مستخدمي مؤسّسته فقط (عزل في كل الاستعلامات). وراثة `schoolId` عند الإنشاء.

**لوحة المدير:** المستخدمون في عرض **شجري** (تبويب الطلاب: مؤسّسة←صفّ←طلاب؛ تبويب المدرّسين/المدراء: مؤسّسة←أعضاء للمدير العام، أو حسب الدور لمدير المدرسة — `UsersTree`)، إنشاء مدرّس/مدير/طالب، المدارس، **المواد والصفوف متداخلة** (الصفّ←مواده مع تحرير/حذف لكل عنصر — `AcademicsTree`، للمدير العام)، **الحقول المخصّصة** (`CustomFieldDef` + `User.customData`: المدير العام يعرّف حقولاً إجبارية/اختيارية لكل جمهور)، استيراد طلاب خارجيين من **Excel فقط** (exceljs) + تصدير بيانات الدخول xlsx.

**لوحة المدير العام (الإشراف عبر المؤسّسات):** صفحة نظرة عامة (`/admin/overview`: إحصاءات المنصّة + جدول لكل مؤسّسة)، تصفّح **البنك العام** (`/admin/quizzes` و`/admin/questions`: قراءة فقط، فلترة بالمؤسّسة/المادة/النوع أو الحالة + ترقيم). النسبة للمؤسّسة عبر `creator.schoolId`.

**ولي الأمر:** دور `PARENT` + جدول ربط `ParentLink` (متعدّد-لمتعدّد: ولي ← عدّة أبناء، وطالب ← عدّة أولياء). **المدير** ينشئ الوليّ ويربطه بأبنائه بلصق رموز الطلاب (بعزل المؤسّسة) من `/admin/parents` (قائمة/إنشاء/إدارة روابط). **لوحة الوليّ** `/parent`: أبناؤه ونتائج كلّ ابن (قراءة فقط) عبر `SessionReviewView` المشترك — مع **فحص ملكية صارم** (`parentOwnsStudent`) لكل وصول لبيانات طالب، وإخفاء النتيجة ما دامت بانتظار التصحيح. الدخول الموحّد يعمل للوليّ (بريد/اسم).

**الاختبار الورقي/المرفوع (تخزين داخل Postgres):** نوع اختبار موازٍ (`Quiz.isFileBased`) بلا أسئلة/عُقد: المدرّس ينشئه ويرفع ورقته (صورة/PDF) من `/teacher/file-exams`، والطالب يرفع صور إجابته من صفحة الأداء ثم يُرسلها (`needsGrading`)، والمدرّس يصحّح يدوياً (درجة كلّية + ملاحظة) من «الإجابات والتصحيح»، ويراها الطالب ووليّ أمره. الملفات في جدول `Attachment` (bytea) خلف طبقة تجريد `src/lib/attachments.ts`، وتُبَثّ عبر `/api/attachments/[id]` **بفحص ملكية صارم فقط** (لا روابط عامّة). حدود: صورة/PDF ≤3MB، 12 صفحة. الدرجة تُخزَّن في `ExamSession.{totalScore,maxPossibleScore,percentage}` + `teacherFeedback`. الرفع عبر `ImageUploadField` المشترك: **ضغط الصورة على المتصفّح** (`imageCompress.ts`: تصغير + JPEG) ثم **معاينة قبل التأكيد** (يرى الرافِع الوضوح والأبعاد والحجم، فيرفع أو يحذف ويختار غيرها).

**الإشعارات (داخل التطبيق):** جدول `Notification` + جرس بعدّاد غير المقروء في رأس كل لوحة + صفحة `/notifications` (تُعلَّم مقروءةً عند الفتح). يُنشَأ إشعار للطالب عند **أوّل تصحيح** لورقته (لا يتكرّر عند التعديل).

**تعليقات المدرّس على الصورة:** جدول `Annotation` (إحداثيات نسبية x,y على صورة الإجابة) — المدرّس ينقر الصورة فيضع دبّوساً بتعليق من «الإجابات والتصحيح» (`ImageAnnotator` editable)، ويراها الطالب ووليّ أمره دبابيس فوق الصورة (قراءة فقط). التعليقات على الصور لا PDF.

**أدلّة الاستخدام:** زرّ «كيف أستخدم صفحتي؟» في رأس كل لوحة يفتح `/guide/[role]` (تبويب جديد) — دليل مخصّص لكل دور (طالب/مدرّس/مدير/وليّ)، محتواه في `src/lib/guide.ts`.

**إتاحة وصول:** متحكّم تكبير النصّ (يُحفظ على الجهاز)، زرّ «الرئيسية» في كل صفحة. تواريخ بحقل أرقام خاصّ (`DateTimeField`) + عرض `formatDateTime` داخل `‹bdi›`.

**إضافات المخطط بعد الأساس:** `Unit`، `School`، `CustomFieldDef`، **`ParentLink`**، **`AppSetting`**، **`Attachment`** (+`AttachmentKind`، `Quiz.isFileBased`، `ExamSession.{needsGrading,teacherFeedback}`)، **`Notification`**، **`Annotation`** (+ قيمة `PARENT` في `Role` وعلاقتا `User.{parentLinks, studentParents}`)؛ وحقول: `User.{isSuperAdmin, schoolId, createdById, customData}`, `Chapter.unitId`, `Quiz.{accessCode, allowCodeJoin}`, `QuizAssignment.extraAttempts`, `StudentProfile.{fatherName, motherName, address, isExternal}`, `StudentAnswer.needsReview`, `User.email` صار اختيارياً.

> البذرة الحالية (`prisma/seed.ts`) **تصفّر كل شيء** ثم تنشئ المدير العام + صفّ «بكالوريا علمي» + 9 مواد (العلمية مُشجَّرة). استخدم Git دائماً.

## أوامر التشغيل
- `npm install` ثم `.env` (DATABASE_URL لـ Neon + AUTH_SECRET طويل).
- `npm run db:push` ← `npm run db:seed` ← `npm run dev` → http://localhost:3000
- **بعد أي تغيير مخطط:** أوقف `dev` ثم `npx prisma generate` ثم `npm run dev` (الخادم يقفل محرّك Prisma).
- الحساب الوحيد بعد البذرة: **المدير العام** `admin@example.com` / `Admin@123` (بقية الحسابات تُنشأ من اللوحة).
