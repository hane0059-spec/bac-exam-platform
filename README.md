# منصة الاختبارات الإلكترونية — الخطوة 1 (الأساس)

تطبيق Next.js 14 واحد (واجهة + API)، عربي RTL، مصادقة JWT عبر `jose` في كوكي httpOnly،
ولوحات تحكّم مبدئية لكل دور مع مخاطبة حسب الجنس.

## المتطلبات
- Node.js 18 أو أحدث
- قاعدة بيانات PostgreSQL (Neon المجانية)

## الإعداد (Windows / PowerShell)

1) ثبّت الحزم:
```powershell
npm install
```

2) أنشئ ملف `.env` (بترميز ASCII لتفادي مشاكل الترميز على Windows):
```powershell
Set-Content -Path .env -Encoding ascii -Value @'
DATABASE_URL="postgresql://USER:PASSWORD@HOST/DB?sslmode=require"
AUTH_SECRET="ضع-هنا-سلسلة-عشوائية-طويلة-32-حرفاً-فأكثر"
'@
```
> ملاحظة: استبدل قيمة `DATABASE_URL` برابط Neon الخاص بك. لا تشارك هذا الرابط مع أحد.

3) ادفع المخطط إلى قاعدة البيانات:
```powershell
npm run db:push
```

4) عبّئ البيانات التجريبية (الصفوف + مادة + 3 حسابات):
```powershell
npm run db:seed
```

5) شغّل التطوير:
```powershell
npm run dev
```
ثم افتح: http://localhost:3000

## الحسابات التجريبية
| الدور   | البريد                 | كلمة السر    |
|---------|------------------------|--------------|
| مدير    | admin@example.com      | Admin@123    |
| مدرّسة  | teacher@example.com    | Teacher@123  |
| طالب    | student@example.com    | Student@123  |

## ما الذي يجب أن تجرّبه
- الدخول بكل حساب → تُوجَّه تلقائياً إلى لوحته.
- لاحظ المخاطبة حسب الجنس: «مرحباً بك» للمدير والطالب، «مرحباً بكِ» للمدرّسة، ومسمّى الدور (مدير/مدرّسة/طالب).
- جرّب فتح مسار دور آخر يدوياً (مثلاً ادخل كطالب ثم اذهب إلى `/admin`) → يُعاد توجيهك إلى لوحتك.
- جرّب فتح `/student` دون تسجيل دخول → تُحوَّل إلى صفحة الدخول.
- تسجيل الخروج يمسح الجلسة.

## معمارية مثبّتة
تطبيق Next.js واحد — لا Docker، لا Redis، لا Nginx. Postgres عبر Prisma.
المصادقة JWT عبر `jose`. الجنس على نموذج `User` ومُضمَّن في الجلسة.
